Adopting the SGDS incrementally

SGDS helps the government design and build better online experiences for the public. As a toolkit to design and build government websites, the SGDS provides:

Framework

A framework that works right out of the box with most of the configuration already done

Code

Our code provides an accessible, mobile-friendly experience across government sites. Youâ€™ll be able to copy and paste code snippets or import only dependencies that you need.

Design assets

By providing the SGDS in Sketch and Figma, we allow designers to hit the ground running by having everything that they need already setup

Guidelines for components

We provide guidelines for user experience for our components and patterns work as intended

Guides

Guides for best practices are provided for designers, developers and writers to help them get started with their digital products

With all these in place, even before using the framework or adding any of the SGDS code, you can start using SGDS by adopting our UX guidelines for the components (e.g guidelines for our text field). When you are ready, you can start incorporating the SGDS code incrementally by customising with Sass so you can pull only the components that you need.

The SGDS is build to be flexible and you can add on components or customise styles if you so wish to and adapt it to the product or digital service you are creating
